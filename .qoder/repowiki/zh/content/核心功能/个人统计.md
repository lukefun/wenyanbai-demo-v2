## 简介
本文件面向"个人统计"功能，围绕学习进度统计、成就等级系统、学习效果评估机制进行系统化说明。重点解释掌握字数统计、正确率计算、等级晋升规则与排名系统设计；阐述统计数据的采集方式、存储格式、展示逻辑与动态更新机制；并给出实现分析，包括 master 计算、等级徽章视觉设计、进度条动画效果、成就徽章获取条件等。**新增**了详细的统计覆盖系统，包括学习详情、准确率详情、测试次数详情和掌握字符详情的交互式覆盖板系统。最后提供学习数据分析方法、进步趋势跟踪与学习计划制定建议。

## 核心组件
- 统计状态与持久化
  - 统计对象：包含总答题数与正确次数，用于计算正确率。
  - 状态对象：记录每张卡片的学习轮次、下次复习时间与正确次数，支持间隔重复算法。

## 详细组件分析

### 掌握字数统计（mastered）
- 统计口径
  - 以"核心字"为单位，统计每个字的所有含义中被正确掌握的比例。
  - 当某字的正确含义占比达到或超过阈值（≥80%）时，视为该字已掌握。
- 实现要点
  - 遍历所有已掌握卡片，按字聚合计数。
  - 对每个字，计算其正确含义数与总含义数的比值。
  - 返回达到阈值的字集合长度作为"已掌握核心字数"。

```mermaid
flowchart TD
Start(["函数入口"]) --> Init["初始化字级计数字典"]
Init --> LoopCards{"遍历已掌握卡片"}
LoopCards --> |是| Group["按字分组累计正确数"]
LoopCards --> |否| Calc["计算每个字的正确比例"]
Calc --> Threshold{"比例 ≥ 80% ?"}
Threshold --> |是| Add["加入已掌握字列表"]
Threshold --> |否| Skip["跳过"]
Add --> Next["下一个字"]
Skip --> Next
Next --> Done{"遍历结束？"}
Done --> |否| Calc
Done --> |是| Return["返回已掌握字数"]
```

## 性能考量
- 渲染优化
  - 使用过渡动画控制进度条宽度变化，避免频繁重排。
  - 字级进度条采用点状标记，减少 DOM 数量。
  - **详情弹窗：** 采用懒加载和虚拟滚动优化大数据集渲染。
- 计算复杂度
  - 掌握字统计对已掌握卡片进行聚合，整体复杂度与卡片数线性相关。
  - 等级匹配通过一次遍历确定区间，常数级开销。
  - **详情分析：** 各详情函数独立计算，避免重复数据处理。
- 存储与同步
  - 本地存储读写在关键节点触发，避免过度频繁写入。
  - **详情数据：** 通过内存缓存减少重复计算。

## 故障排查指南
- 统计异常
  - 现象：正确率不更新或数值异常。
  - 排查：确认测验与小测流程中的统计更新是否触发；检查本地存储键值是否存在。
- 掌握字数不变化
  - 现象：核心字掌握数长期不变。
  - 排查：确认"记住了/还不熟"操作是否导致卡片状态更新；检查阈值与字级聚合逻辑。
- 等级不晋升
  - 现象：已掌握字数增长但等级不变。
  - 排查：确认等级区间配置与当前掌握字数匹配逻辑；检查进度条宽度计算。
- 进度条不更新
  - 现象：进度条宽度不随学习进度变化。
  - 排查：确认首页与"我的"页的进度条宽度设置逻辑；检查样式过渡是否生效。
- **详情弹窗问题**
  - 现象：统计详情无法打开或显示空白。
  - 排查：确认DOM元素存在性；检查详情构建函数的逻辑；验证数据完整性。

## 结论

## 附录

### 学习数据分析方法与建议
- 进度趋势
  - 周/月维度对比"已掌握核心字数"与"正确率"，识别稳定提升与波动期。
  - **利用详情弹窗：** 通过准确率详情分析薄弱环节，针对性强化训练。
- 错误分析
  - 分析测验与小测中的错误选项分布，定位易混淆含义，针对性强化。
  - **详情分析：** 使用准确率详情和测试次数详情识别高频错误模式。
- 计划制定
  - 设定阶段性目标（如"掌握 X 字""正确率达到 Y%"），结合等级进度条进行里程碑管理。
  - 利用"待复习/新词"筛选，优先安排复习任务，确保间隔重复节奏稳定。
  - **掌握分析：** 通过掌握字符详情识别学习瓶颈，调整学习策略。
